# GlitchTip Self-Hosted Stack

Production-grade error tracker for Bike Trip Planner. See [ADR-031](../../docs/adr/adr-031-error-tracking-strategy.md) for the rationale.

## Architecture

Four containers on an isolated Docker network:

| Container | Image | Role |
| --- | --- | --- |
| `glitchtip-postgres` | `postgres:18-alpine` | Dedicated PostgreSQL (not shared with the app). |
| `glitchtip-redis` | `redis:7-alpine` | Celery broker + cache. |
| `glitchtip-web` | `glitchtip/glitchtip:latest` | HTTP UI + ingestion API (published on `:8000`). |
| `glitchtip-worker` | `glitchtip/glitchtip:latest` | Celery worker + beat (event ingestion, retention sweeper). |

## Coolify Installation

1. In Coolify, **+ New Resource → Docker Compose**.
2. Source: paste the contents of `.docker/glitchtip/docker-compose.yml` (or point to this path in the repository).
3. Configure the following environment variables (Coolify "Environment Variables" tab):

   | Variable | Required | Notes |
   | --- | --- | --- |
   | `GLITCHTIP_DB_PASSWORD` | yes | Generated by Coolify, then re-used by both `glitchtip-postgres` and the worker/web containers. |
   | `GLITCHTIP_SECRET_KEY` | yes | Django `SECRET_KEY`, 50+ random bytes (`openssl rand -base64 50`). |
   | `GLITCHTIP_DOMAIN` | yes | `https://errors.biketrip.mooo.com` |
   | `GLITCHTIP_FROM_EMAIL` | no | Defaults to `noreply@biketrip.mooo.com`. |
   | `GLITCHTIP_EMAIL_URL` | no | Set to an SMTP URL (`smtp://user:pass@host:587`) to enable email notifications. Defaults to `consolemail://` (logs to stdout). |

4. Set the Coolify **Domain** for the `glitchtip-web` container to `errors.biketrip.mooo.com`. Traefik will auto-provision a Let's Encrypt certificate.
5. Deploy.
6. Browse to `https://errors.biketrip.mooo.com`. The first account you create automatically becomes the superuser.

## DNS

The sub-domain `errors.biketrip.mooo.com` lives on FreeDNS:

```
errors.biketrip.mooo.com   CNAME   biketrip.mooo.com
```

(Or an `A` record pointing at the Oracle VM's public IP — depending on the FreeDNS setup. Either works; the CNAME is preferred to avoid two records to update on IP change.)

## Wiring the SDKs

### Backend (`api/.env.local.prod`)

```
SENTRY_DSN=https://<public_key>@errors.biketrip.mooo.com/<project_id>
APP_RELEASE=<commit_sha>
```

`<public_key>` and `<project_id>` come from the GlitchTip project settings ("Client Keys (DSN)").

### Frontend (`pwa/.env.production`)

```
NEXT_PUBLIC_SENTRY_DSN=https://<public_key>@errors.biketrip.mooo.com/<project_id>
NEXT_PUBLIC_APP_RELEASE=<commit_sha>
SENTRY_AUTH_TOKEN=<glitchtip_auth_token>   # required for source-map upload at build time
```

Generate the auth token from GlitchTip → "Auth Tokens" with scope `project:releases` + `project:write`.

## Resource Budget

Total: ~500 MB RAM, ~2 GB persistent disk over the 30-day retention window.

- `glitchtip-web` ~150 MB
- `glitchtip-worker` ~200 MB
- `glitchtip-postgres` ~80 MB (working set)
- `glitchtip-redis` ~50 MB

## Backups

Add the `glitchtip_postgres_data` volume to the Coolify backup schedule. Event data can be regenerated by re-ingesting from logs in a pinch, but issue triage history (assignments, comments, resolution status) cannot.
